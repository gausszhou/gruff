# ANSI 代码生成

## SGR 常量

| 常量 | 代码 | 效果 |
|---|---|---|
| `ansiReset` | `\x1b[0m` | 完全重置（仅在 `Style.end()` 中作为回退使用） |
| `ansiBold` | `\x1b[1m` | 粗体 / 高亮 |
| `ansiItalic` | `\x1b[3m` | 斜体 |
| `ansiUnderline` | `\x1b[4m` | 下划线 |
| `ansiNoBold` | `\x1b[22m` | 取消粗体 |
| `ansiNoItalic` | `\x1b[23m` | 取消斜体 |
| (内联) | `\x1b[24m` | 取消下划线 |
| (内联) | `\x1b[39m` | 默认前景色 |
| (内联) | `\x1b[49m` | 默认背景色（或文档背景色） |

## 样式协议

每个样式元素在内容之前调用 `Style.start()`，之后调用 `Style.end(bg)`。

### `start()`

1. 粗体 → `\x1b[1m`
2. 斜体 → `\x1b[3m`
3. 下划线 → `\x1b[4m`
4. 前景色 → `\x1b[38;...m`
5. 背景色 → `\x1b[48;...m`

### `end(bg Color)`

1. 斜体 → `\x1b[23m`
2. 粗体 → `\x1b[22m`
3. 下划线 → `\x1b[24m`
4. 前景色 → `\x1b[39m`（始终重置为默认前景色）
5. 背景色 → `\x1b[48;2;R;G;Bm`（恢复文档背景色）或 `\x1b[49m`（终端默认）

**为什么使用特定的撤销代码而不是 `\x1b[0m`?** 考虑嵌套样式：

```
\x1b[1m粗体 \x1b[3m斜体加粗体\x1b[23m仅粗体\x1b[22m普通
```

在内层关闭时使用 `\x1b[0m` 也会重置外层的粗体状态。通过使用特定的撤销代码，每个样式层都可以独立管理。

## 嵌套示例

```
**粗体**          → \x1b[1m粗体\x1b[22m
*斜体*            → \x1b[3m斜体\x1b[23m
***粗斜体***      → \x1b[3m\x1b[1m粗斜体\x1b[22m\x1b[23m
**粗体 `代码`**   → \x1b[1m粗体 \x1b[48;5;236;38;5;15m代码\x1b[39m\x1b[48;2;20;20;20m\x1b[22m
```

## 表格单元格背景隔离

渲染每个单元格内容后，渲染器会输出：

```
\x1b[39m           ← 重置前景色
\x1b[48;2;R;G;Bm  ← 恢复文档背景色
```

这可以防止内联代码背景色扩散到单元格的填充区域或分隔符 `│`。
